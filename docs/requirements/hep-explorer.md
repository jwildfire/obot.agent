# hep-explorer requirements matrix

> Source-backed requirement matrix for the safety.viz **hep-explorer** (eDISH) module, harvested from the original [SafetyGraphics/hep-explorer](https://github.com/SafetyGraphics/hep-explorer) renderer (v1.4.2) cross-referenced with the safety.viz port, plus the baseline-referenced **composite plot** (`HEP-COMP-*`, [safety.viz#67](https://github.com/jwildfire/safety.viz/issues/67)). Treat this as a reviewed starting matrix: the shipped rows are AI-reviewed and awaiting human sign-off; the deferred rows are filed as v1.3.0 follow-up sub-issues. Requirement IDs use the module's condensed `HEP-*` scheme, matching the IDs cited in the source JSDoc, the tests, and the port's [`docs/hep-explorer-coverage.md`](https://github.com/jwildfire/safety.viz/blob/main/docs/hep-explorer-coverage.md). IDs are the **base** form (`HEP-QUAD-001`, not `-001A`/`-001B`) so each row's ID resolves the matching `requirementIds` tag in the shipped `docs/evidence/hep-explorer/evidence.json`, which the evidence pages look up by exact ID; where a source requirement was split during review, the atomic parts are merged into the single base-ID row. The `Evidence Type` column routes rows: `unit` → Vitest (`tests/unit/hep-explorer/`), `browser` → Playwright (`tests/e2e/hep-explorer.spec.js`), `deferred` → filed as a v1.3.0 sub-issue (not yet implemented). Where a source row is covered by both suites, the browser evidence is listed.

The baseline-referenced **composite plot** (safety.viz [#67](https://github.com/jwildfire/safety.viz/issues/67), PR [safety.viz#69](https://github.com/jwildfire/safety.viz/pull/69)) is a faithful port of the FDA Composite-eDISH-Plot reference (`Composite_eDISH_Model.R`, public domain / MIT, DOI [10.5281/zenodo.10892050](https://doi.org/10.5281/zenodo.10892050)) implementing Tesfaldet et al., "Composite Plot for Visualizing Aminotransferase and Bilirubin Changes in Clinical Trials of Subjects with Abnormal Baseline Values," Drug Safety 2024;47:699–710. It anchors each subject to their OWN baseline (×BLN) alongside the population reference (×ULN) for the abnormal-baseline population where the standard ×ULN eDISH view both raises false alarms and masks real on-treatment change. Requirement text below is sourced from the ported R algorithm (`src/hep-explorer/composite.js`) rather than reverse-engineered from the paper.
