# participant-profile requirements matrix

> Requirement matrix for the safety.viz **participant-profile** module — the standardized-lab participant drill-down (header / labs-over-time spaghetti / measure table with sparkline insets) extracted from the original [SafetyGraphics/hep-explorer](https://github.com/SafetyGraphics/hep-explorer) Details View into a chart-agnostic module with two mounts: standalone (`SafetyViz.participantProfile`) and docked below a host chart. Rows are authored from the nine issue-level requirements PPRF-1..PPRF-9 of [safety.viz#98](https://github.com/jwildfire/safety.viz/issues/98), decomposed into atomic testable rows. Requirement IDs use the `PPRF-<AREA>-<NNN>` scheme in **base** form (`^[A-Z]{2,4}-[A-Z]+-\d+[A-D]?$`), matching the IDs tagged in the module's tests so each row resolves the matching `requirementIds` in the shipped `docs/evidence/participant-profile/evidence.json` by exact base-ID lookup. The `Evidence Type` column names the suite that evidences the row: `unit` → Vitest (`tests/unit/participant-profile/`), `browser` → Playwright (`tests/e2e/participant-profile.spec.js`, `tests/e2e/hep-explorer.spec.js`). Rows carry Status `planned` until the implementation PR merges and `docs/participant-profile-coverage.md` lists them; the `Test/Evidence Link` points at the implementation issue until then.

## Requirement context

- Requirement: [obot.roadmap#45](https://github.com/jwildfire/obot.roadmap/issues/45) — participant-profile module (the "A+D hybrid" of the [design options report](https://jwildfire.github.io/obot.roadmap/reports/participant-profile-module-options-2026-07-22/))
- Implementation: [safety.viz#98](https://github.com/jwildfire/safety.viz/issues/98) — standalone renderer + dock, hep-explorer adopts first; `PPRF-HDR-003` additionally closes [safety.viz#53](https://github.com/jwildfire/safety.viz/issues/53) (templated `participantProfileURL` link-out)
- Prerequisite: [safety.viz#91](https://github.com/jwildfire/safety.viz/issues/91) — hep-core split (the module consumes `hep-core` reducers directly; design decision D4: no interim adapter)
- Out of scope here: [safety.viz#99](https://github.com/jwildfire/safety.viz/issues/99) — rollout to the six remaining lab-family adopters; exposure strip, AE/conmed tracks, drawer mode (v2 candidates per the design report); `Widget_ParticipantProfile` in gsm.safety
- Issue-level mapping: PPRF-1 → `PPRF-CORE-001/002/003`; PPRF-2 → `PPRF-HDR-001/002/003`; PPRF-3 → `PPRF-SPAG-*`; PPRF-4 → `PPRF-TBL-*`; PPRF-5 → `PPRF-STEP-*`; PPRF-6 → `PPRF-EVT-001`; PPRF-7 → `PPRF-HEP-*`; PPRF-8 → `PPRF-ACC-001`; PPRF-9 → `PPRF-GATE-001`
