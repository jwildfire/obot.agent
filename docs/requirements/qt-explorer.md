# qt-explorer requirements matrix

> Source-backed requirement matrix for the safety.viz **qt-explorer** (QT Safety Explorer) module — a Chart.js reimplementation of the original [SafetyGraphics/qtexplorer](https://github.com/SafetyGraphics/qtexplorer), **Phase 1** ([safety.viz#68](https://github.com/jwildfire/safety.viz/issues/68); parent requirement [obot.roadmap#36](https://github.com/jwildfire/obot.roadmap/issues/36)). Requirement IDs use the module's condensed `QT-*` scheme cited in the source JSDoc, the tests, and the port's [`docs/qt-explorer-coverage.md`](https://github.com/jwildfire/safety.viz/blob/main/docs/qt-explorer-coverage.md). IDs are the **base** form so each row resolves the matching `requirementIds` tag in the shipped `docs/evidence/qt-explorer/evidence.json`, which the evidence pages look up by exact ID. The `Evidence Type` column routes rows: `unit` → Vitest (`tests/unit/qt-explorer/`), `browser` → Playwright (`tests/e2e/qt-explorer.spec.js`); where a row is covered by both suites the browser evidence is listed. Treat this as a reviewed starting matrix: the rows are AI-reviewed and awaiting human sign-off.

## Requirement context

- Requirement: [obot.roadmap#36](https://github.com/jwildfire/obot.roadmap/issues/36) — QT Safety Explorer, Phase 1 (central tendency + outlier scatter + categorical exceedance)
- Implementation: [safety.viz#68](https://github.com/jwildfire/safety.viz/issues/68) — released in safety.viz v1.4.0
- Phase 2 (out of scope here): [obot.roadmap#37](https://github.com/jwildfire/obot.roadmap/issues/37) — QTcI individualized correction, PR/QRS/JT multi-interval rows, per-subject drill-down, QT-RR hysteresis, the guided ICH-E14 workflow, and a moxifloxacin positive-control arm. None of these are present in the CDISC Pilot ADEG demo and none are represented below.

## Scope

**Phase 1 (this matrix):** per-arm central-tendency change over time with a mean/median statistic and a Δ / placebo-corrected ΔΔ toggle, a two-sided 90% CI band, the mode-labelled reference line, the peak-effect-visit marker, and the ICH-E14 metric; the outlier scatter of change-from-baseline vs baseline QTc with the 450/480/500 ms absolute diagonals, the 30/60 ms change lines (per-visit mode), the zero no-change line, per-arm colour and mark shape, and the point tooltip; the by-arm categorical exceedance table for the absolute and change thresholds with per-arm denominators; and the view / correction / statistic / display / timepoint / filter controls. Corrections are **QTcF / QTcB** plus **heart rate**. The demo data is real **CDISC Pilot ADEG** (built from `pharmaverseadam`), vendored as `site/data/adeg.csv`.

The ΔΔ CI is a large-sample normal approximation for exploratory screening, **not** the regulatory ANCOVA/MMRM least-squares-means bound; the graphic flags signals for review, it does not confirm a QT-prolongation conclusion (which requires the validated ICH-E14 analyses).

## Source inventory

- `SafetyGraphics/qtexplorer` — the original QT Safety Explorer renderer whose views this port reimplements
- safety.viz port: `src/qt-explorer/` (`configure`, `checkInputs`, `structureData`, `getScales`, `getPlugins`), `src/qt-explorer.js`, `src/data/schema/qt-explorer.json`
- ICH-E14 guidance — the ~10 ms threshold of regulatory concern and the largest-upper-bound-of-the-two-sided-90%-CI metric; the CSRC clinical workflow for the 30/60 ms change and 450/480/500 ms absolute thresholds
- safety.viz tests: `tests/unit/qt-explorer/` (`configure`, `checkInputs`, `structureData`, `getScales`, `getPlugins`, `export`), `tests/e2e/qt-explorer.spec.js`
- safety.viz coverage doc: `docs/qt-explorer-coverage.md`
